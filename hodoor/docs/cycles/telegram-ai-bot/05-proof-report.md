# Proof — telegram-ai-bot

## What was implemented

### Files created

| File | Purpose |
|------|---------|
| `bot/__init__.py` | Package marker |
| `bot/config.py` | `BotConfig` dataclass + `load_config()`, validates TELEGRAM_TOKEN and OPENAI_API_KEY at startup |
| `bot/rate_limiter.py` | Thread-safe sliding-window rate limiter keyed by Telegram user ID (default 20/min) |
| `bot/llm.py` | `get_response(text, config)` — calls OpenAI gpt-4o, returns user-friendly strings on all error conditions (RateLimitError, APIConnectionError, APIError, unexpected) |
| `bot/handlers.py` | Async handlers for text (sends to LLM), voice, photo, video (placeholder acknowledgement); each checks rate limit first |
| `bot/main.py` | Entry point: loads config, builds Telegram Application, registers all handlers, starts polling |
| `pyproject.toml` | Project metadata + dependencies: python-telegram-bot>=21.0, openai>=1.30.0 |
| `uv.lock` | Reproducible lockfile generated by uv |
| `Dockerfile` | python:3.12-slim, installs uv, runs `uv pip install --system .`, launches `python -m bot.main` |
| `.dockerignore` | Excludes .git, .env, tmp/, docs/, pyc files |

### Files modified

| File | Change |
|------|--------|
| `docker-compose.yml` | Added `telegram-bot` service with build context, env_file (.env), restart: unless-stopped |

### Behaviour produced

- Text messages: forwarded to OpenAI gpt-4o, reply sent back to user
- Voice/photo/video messages: acknowledged with a placeholder ("Received your voice/photo/video. Processing will be available in a future update.")
- Rate limiting: per-user sliding window, 20 messages/minute by default (configurable via RATE_LIMIT_PER_MINUTE)
- Error resilience: all OpenAI error types return a user-friendly message and log the error; the bot never crashes on LLM failures
- Config validation: bot exits with a clear RuntimeError at startup if TELEGRAM_TOKEN or OPENAI_API_KEY is missing

## How to verify

1. Add to `.env` (create it at repo root if absent):
   ```
   TELEGRAM_TOKEN=your_bot_token_from_BotFather
   OPENAI_API_KEY=your_openai_api_key
   ```

2. Start the stack:
   ```bash
   docker compose up --build telegram-bot
   ```

3. Open Telegram, find your bot, send a text message. You should receive an AI response within ~5 seconds.

4. Send a voice message, photo, and video. Each should receive a placeholder acknowledgement without crashing.

5. To test rate limiting locally (without Docker):
   ```bash
   TELEGRAM_TOKEN=fake OPENAI_API_KEY=fake uv run python -c "
   from bot.rate_limiter import RateLimiter
   rl = RateLimiter(max_per_minute=3)
   print([rl.is_allowed(42) for _ in range(4)])  # [True, True, True, False]
   "
   ```

## Known gaps

- No conversation history: every message is stateless, intentionally per brief.
- Media files are acknowledged but not processed: voice transcription/image vision deferred to future cycle.
- Rate limiter is in-memory: resets on container restart. Acceptable for single-user scope.
- No `/start` or `/help` command handlers: not in scope per brief.
